## What is this?

Most LLM resources are written for researchers, not developers. This is a no-jargon, no-fluff glossary built for backend engineers, full-stack devs, and platform engineers who are entering the AI/LLM space and need clear, practical explanations — not academic abstractions.

Every term includes:
- A one-line definition
- A plain-English explanation (max ~150 words)
- A real-world example grounded in actual dev scenarios (Laravel, APIs, Indian product context)
- Related terms for navigation

- Real-world example must be practical, not theoretical
- No academic jargon — if a term needs another term to explain it, explain that term too
- Analogies grounded in everyday dev work preferred
